# DecisionLog — 주요 의사결정 기록

## D-001 (2026-07-14) 기반 기술: Claude API 직접 구현

- **결정**: Messages API + tool use 위에 에이전트 루프와 오케스트레이션을 직접 구현한다.
- **대안**: (a) Claude Agent SDK — Claude Code 하네스를 라이브러리로 사용,
  (b) Managed Agents — Anthropic이 루프·샌드박스 호스팅(베타, multiagent coordinator 내장).
- **근거**: 오케스트레이션 자체가 이 프로젝트의 핵심 산출물이므로 통제력이 최대인 층을 선택.
  메시지 패싱 구조(D-003)는 자체 구현이 필요해 기성 하네스의 이점이 줄어듦.
- **결정자**: 사용자.
- **수정**: [D-009](#d-009-2026-07-14-llm-계층-멀티-프로바이더-추상화)에서 일부 수정 —
  "직접 구현" 원칙은 유지하되, LLM 호출 계층은 Claude 전용이 아닌 멀티 프로바이더로 확장.

- **결정자**: 사용자.

## D-003 (2026-07-14) 오케스트레이션 패턴: 자율 협업 (메시지 패싱)

- **결정**: 중앙 조율자가 태스크를 분배하는 방식이 아니라, 에이전트들이 메시지 버스를 통해
  서로 직접 메시지를 주고받으며 협업하는 구조로 한다.
- **대안**: 오케스트레이터-워커, 파이프라인(역할 체인).
- **근거**: 사용자 선택. 유연성이 가장 높은 패턴. 단, 수렴 제어가 어려우므로
  종료 정책(결과 제출 도구 + 예산 상한 + 유휴 감지)을 시스템 차원에서 강제한다(Plan M2).
- **결정자**: 사용자.

## D-004 (2026-07-14) 처리 대상: 범용 태스크

- **결정**: 특정 도메인(코딩/리서치 등)에 고정하지 않고, 팀 구성(역할 프롬프트)을
  설정 파일로 정의해 어떤 태스크든 처리할 수 있게 한다.
- **근거**: 사용자 선택. 팀 구성이 데이터(설정)로 분리되어 있으면 도메인 특화 팀은
  설정 추가만으로 파생 가능.
- **결정자**: 사용자.

## D-005 (2026-07-14) 인터페이스: 로컬 서버 + 웹 대시보드

- **결정**: FastAPI 기반 로컬 서버가 세션을 관리하고, 웹 대시보드에서 태스크 제출·
  메시지 흐름 관찰·결과 확인을 한다.
- **근거**: 사용자 선택. 메시지 패싱 구조는 흐름이 비선형이라 시각적 관측성이 중요.
- **결정자**: 사용자.

## D-006 (2026-07-14) 서버 스택: FastAPI + uvicorn + SSE

- **결정**: 서버는 FastAPI + uvicorn, 실시간 스트림은 SSE(Server-Sent Events)로 한다.
- **대안**: Flask(동기라 부적합), aiohttp(저수준), WebSocket(양방향 필요성 낮음).
- **근거**: asyncio 네이티브(요청 경로 블로킹 금지 제약과 부합), 자동 문서화,
  SSE는 단방향 이벤트 스트림에 충분하고 구현이 단순. 대시보드→서버 입력은 REST로 충분.
- **결정자**: Claude 제안, 사용자 이의 없으면 유지. (변경 원하면 이 항목 갱신)

## D-008 (2026-07-14) 기본 모델: OpenAI gpt-5.6 Terra (D-007 번복)

- **결정**: 에이전트 기본 모델을 OpenAI **GPT-5.6 Terra** 티어로 변경한다.
  에이전트별 모델 오버라이드(팀 설정)는 D-007과 동일하게 유지.
- **근거**: 사용자 결정. ChatGPT subscription 연동을 전제로 함.
- **사실 확인 (2026-07-14 웹 조사, Research.md §6)**:
  - GPT-5.6은 2026-07-09 출시된 3티어 패밀리 — Sol($5/$30), Terra($2.50/$15),
    Luna($1/$6, $/1M tokens). Terra는 GPT-5.5급 성능의 중간 티어. 티어 선택은 사용자(Terra).
  - ChatGPT 구독과 API 키 과금은 **별개 시스템**이므로 구독으로 일반 API 호출은 불가.
    단 "Sign in with ChatGPT"(OAuth, BYOS) 경로로 사용자 구독 quota 사용이 가능하다는
    조사 결과 — 자체 앱에서의 실현 가능성은 `확실하지 않음`, M2 착수 전 스파이크로 검증
    (실패 시 `OPENAI_API_KEY` 과금 폴백 후 이 항목 갱신).
  - 정확한 API 모델 ID(`gpt-5.6-terra` 추정)는 `확실하지 않음` — 구현 시 공식 문서 확인.
- **결정자**: 사용자.

## D-009 (2026-07-14) LLM 계층: 멀티 프로바이더 추상화

  계약은 M1에서 확정하고, **OpenAI 어댑터를 우선** 구현, Anthropic 어댑터는 후순위.
  팀 설정에서 에이전트별로 GPT/Claude 혼용 가능하게 한다.
- **대안**: OpenAI 단일(추상화 없음) — 단순하지만 Claude 혼용 시 재작업 필요.
- **근거**: 기본 모델이 GPT-5.6으로 바뀌며(D-008) D-001의 "Claude Messages API 직접 구현"

  추상화. 아직 코드가 없어 전환 비용 0. "계약 우선" 개발 지침과도 부합.
